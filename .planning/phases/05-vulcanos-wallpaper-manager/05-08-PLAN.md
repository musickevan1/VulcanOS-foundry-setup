<task type="checkpoint:human-verify" gate="blocking">
  <name>Verify complete wallpaper manager functionality</name>
  <what-built>
VulcanOS Wallpaper Manager - a GTK4/Adwaita GUI application for:
- Visualizing multi-monitor layout
- Selecting and applying wallpapers per-monitor
- Saving/loading wallpaper profiles
- Splitting panoramic images across monitors
- Integration with vulcan-menu
  </what-built>
  <how-to-verify>
1. **Build and Launch**
   ```bash
   cd vulcan-wallpaper-manager
   cargo build --release
   ./target/release/vulcan-wallpaper-manager
   ```
   - Window should open with Adwaita styling
   - Header bar shows "Wallpaper Manager" title

2. **Monitor Layout**
   - All connected monitors displayed in top section
   - Monitor positions match actual physical layout
   - Vertical monitor (DP-10) should appear rotated
   - Clicking a monitor highlights it in blue

3. **Wallpaper Selection**
   - Thumbnails appear in bottom grid
   - Thumbnails load from ~/Pictures/Wallpapers/
   - Clicking wallpaper selects it
   - Apply button becomes active when both monitor and wallpaper selected

4. **Apply Wallpaper**
   - Click a monitor
   - Click a wallpaper
   - Click "Apply" button
   - Wallpaper should appear on that monitor immediately

5. **Profile Management**
   - Set wallpapers for multiple monitors
   - Select a profile name from dropdown (e.g., "desktop")
   - Click Save icon
   - Quit and relaunch app
   - Load the profile - wallpapers should be restored

6. **Panoramic Split (if time permits)**
   - Click panoramic import button
   - Select a wide image
   - Enter a name
   - Click "Split & Apply"
   - All monitors should show their portion of the panoramic

7. **Menu Integration**
   - Press Super+Space to open vulcan-menu
   - Navigate to Settings
   - Find "Wallpaper Manager" entry
   - Click it - app should launch
  </how-to-verify>
  <resume-signal>
Type "approved" if all functionality works correctly.
